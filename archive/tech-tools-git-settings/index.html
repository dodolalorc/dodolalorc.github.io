🔑Git环境配置和SSH公钥设置

发布于：2025-08-01 #Git#SSH 共 647 字 约 3 分钟

一般远程仓库都支持使用 ssh 访问和读写数据，提供 ssh 进行连接时，会使用本地的私钥文件进行身份验证，配置 ssh 密钥的步骤即：

  1. 在本地生成 ssh 密钥对
  2. 上传公钥到远程存储库

配置密钥

1. 生成 ssh 密钥对

plaintext
UTF-8|1 Line|
`windows` 用户建议使用PowerShell
Bash
UTF-8|1 Line|
ssh-keygen -t ed25519 -C "SSH Key"
  • -t：key 类型
  • -C：注释

在命令行一路回车（三次）即可，将把密钥放在默认路径下。

2. 查看生成的 SSH 公钥和私钥

Bash
UTF-8|1 Line|
ls ~/.ssh/

默认会将 ssh 密钥放在~路径下的隐藏文件夹.ssh中。输出：、

Bash
UTF-8|1 Line|
id_ed25519  id_ed25519.pub
  • 私钥文件  id_ed25519
  • 公钥文件  id_ed25519.pub

3. 读取公钥

使用 cat 命令，或者直接使用记事本打开文件进行复制：

Bash
UTF-8|1 Line|
cat ~/.ssh/id_ed25519.pub

复制公钥内容后，将其上传到远程存储库中，一般在个人账号设置中会有 ssh 设置栏。

4.测试 ssh 连接

使用ssh -T命令进行测试，其后加在远程存储库的用户名。

Bash
UTF-8|1 Line|
ssh -T git@github.com

正确响应：

Bash
UTF-8|1 Line|
Hi your_username! You've successfully authenticated, but GitHub does not provide shell access.

Git config 信息配置

Git 的配置信息分为三个级别，分别为：

  • --local：仓库级别
  • --global：用户级别
  • --syustem：系统级别

1. 设置全局用户信息

Bash
UTF-8|2 Lines|
git config --global user.name "你的用户名"
git config --global user.email "你的邮箱"

2. 常用全局配置

Bash
UTF-8|14 Lines|
# 设置默认文本编辑器（VSCode为例）
git config --global core.editor "code --wait"

# 设置换行符处理（Windows推荐）
git config --global core.autocrlf true

# 设置换行符处理（Linux/Mac推荐）
git config --global core.autocrlf input

# 启用彩色输出
git config --global color.ui true

# 设置默认分支名称（新版本Git默认是main）
git config --global init.defaultBranch main

3. 查看配置信息

Bash
UTF-8|9 Lines|
# 查看所有配置
git config --list

# 查看全局配置
git config --global --list

# 查看特定配置项
git config user.name
git config user.email

4. 仓库特定配置

如果想将某个特定仓库设置不同的配置（比如将工作邮箱和个人邮箱分开）：

Bash
UTF-8|3 Lines|
cd /path/to/repo
git config user.name "工作用户名"
git config user.email "工作邮箱"

5. 配置别名

配置别名相当于给 git 命令创建“快捷方式”，可以提高工作效率，比如：

  • 把  git checkout  简化为  git co
  • 把  git branch  简化为  git br
  • 把  git commit  简化为  git ci

配置命令格式：

Bash
UTF-8|1 Line|
git config --global alias.新名称 '原命令'

常用别名：

Bash
UTF-8|4 Lines|
git config --global alias.co checkout
git config --global alias.br branch
git config --global alias.ci commit
git config --global alias.st status
